## 4. 胜利庆祝纸屑（本轮新增收尾）

7 款「有胜利态」的新游戏此前无胜利纸屑（仅有 isWin，无 `J.confetti` 调用）。已在各自 `render()` 的 win 分支注入：
```js
if(!confettiFired){ confettiFired=true; if(J&&J.confetti) J.confetti(window.innerWidth/2, window.innerHeight*0.3, {n:120, life:2.4}); }
```
并加 `let confettiFired=false;` 声明，`else` 分支 `confettiFired=false` 自复位（避免通关后重复点击刷屏、且新开局自动就绪）。涉及：`binairo / hitori / yajilin / starbattle / tents / killersudoku / xiangqi`。
> 注：harness 沙箱中 `Juice` 为无 confetti 的 no-op 桩，故单测不受影响；真实浏览器 `Juice.confetti` 存在即触发。

## 5. 关键坑（续前三轮，本轮再次验证）

- **全局 `Math.random` 跨测试共享**：harness 把真实 `Math` 透传给所有游戏。改任何游戏的随机消耗量后必跑全量 `node run.js`（本轮新增 16 谜题生成均用局部 `mk(seed)` PRNG，不污染共享流）。
- **`window.innerWidth/innerHeight` 在沙箱挂在 `window` 上**：裸 `innerWidth` 会 ReferenceError，必须用 `window.innerWidth`。
- **`document` 是真值 Proxy**：渲染不能从 DOM children 回读逻辑状态，动态元素须存 JS 数组。
- **系统 git**：`export PATH="/d/Git/cmd:$PATH"`，自带 PortableGit 有 DNS 线程 bug。
- **禁止 push --force / 跳测试 push / 删 `.workbuddy/`**。
