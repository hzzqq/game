## 3. 统一注入范式（本轮沉淀，可复用）

**胜利/里程碑 confetti**（参考 `arkanoid.html`）：
```js
let _confettiFired = false;            // 独立只读锁，与 Juice 是否真实存在解耦
// 在胜利/通关/里程碑跳变处：
if (!_confettiFired) {
  _confettiFired = true;
  if (typeof Juice !== 'undefined' && Juice.confetti)
    Juice.confetti(window.innerWidth/2, window.innerHeight*0.3, {n:120, life:2.4});
}
// 暴露只读钩子（加到 window.__t）：
confettiFired: () => _confettiFired
// 在 startGame / newGame / reset 里把 _confettiFired 重置为 false
```
- 所有 `Juice.confetti/burst/shake` 必须带 `typeof Juice!=='undefined'` 守卫（harness 的 Juice 桩无 confetti 方法，守卫确保测试不崩）。
- `_confettiFired` 锁防重复触发；新开局自动就绪。
- 无尽类（如 racing）用「里程碑冲线」替代原生胜利态，纯视觉、不结束游戏。

**测试钩子**：游戏内加极小只读标记 `window.__t.confettiFired`（getter/函数），独立于 Juice，使 harness 也能断言；用 false→true 跳变判定。

**难度注入**（参考 `ataxx.html`）：`DIFFICULTY`+`DIFF_ORDER`+`setDifficulty/getDifficulty/setRand`，AI 按 `cfg.aiRandom` 概率随机化，整函数 `document` 守卫。

## 4. 关键坑（续前轮，本轮再次验证）

- **全局 `Math.random` 跨测试共享**：harness 把真实 `Math` 透传给所有游戏。改任何游戏的随机消耗量会移位下游测试。本轮所有新增随机一律走局部 PRNG（`_rand`/mulberry32），newGame/boot 阶段零新增裸 `Math.random()`。
- **子代理空汇报缺陷**：并行派发子代理时曾两次遇到「返回空/假汇报但实际未改文件」（轮7、轮11）。修复：用 `grep -c confettiFired games/<g>.html` 核实 + 重派，最终确认真实落地。本轮收尾也用同法独立核对（25 款 batch4 文件 `confettiFired` 计数均 ≥1）。
- **`window.innerWidth/innerHeight` 在沙箱挂在 `window` 上**：必须用 `window.innerWidth`，裸 `innerWidth` 会 ReferenceError。
- **`document` 是真值 Proxy**：渲染不回读 DOM children，动态元素存 JS 数组。
- **系统 git**：`export PATH="/d/Git/cmd:$PATH"`（自带 PortableGit 有 DNS 线程 bug）；本沙箱无 GitHub 凭据，push 需用户提供 PAT 走内联 credential helper（`GIT_TOKEN` 环境变量，不落盘）。
- **铁律**：禁止 `push --force` / 跳测试 push / 删 `.workbuddy/` / 改核心玩法逻辑做 UI 注入。
